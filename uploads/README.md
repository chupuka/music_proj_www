# Хранилище файлов

Эта папка содержит загруженные файлы для музыкального стримингового приложения.

## Структура

```
uploads/
├── tracks/          # Аудио файлы (MP3, WAV, FLAC и т.д.)
│   ├── {track_id}/
│   │   └── track.mp3
│   └── ...
└── artwork/         # Обложки альбомов и артистов
    ├── albums/
    │   ├── {album_id}.jpg
    │   └── ...
    └── artists/
        ├── {artist_id}.jpg
        └── ...
```

## Использование

### Формат именования файлов

**Треки:**
- Путь: `uploads/tracks/{track_id}/track.{extension}`
- Пример: `uploads/tracks/1/track.mp3`
- В БД: `file_path = "uploads/tracks/1/track.mp3"`

**Обложки:**
- Альбомы: `uploads/artwork/albums/{album_id}.jpg`
- Артисты: `uploads/artwork/artists/{artist_id}.jpg`

## Размеры файлов

- **Аудио файлы:** обычно 3-10 MB (MP3), до 50 MB (FLAC)
- **Обложки:** рекомендуется до 2 MB (JPG/PNG, разрешение 1000x1000)

## Резервное копирование

⚠️ **Важно:** Эта папка содержит пользовательские файлы и должна быть включена в систему резервного копирования.

Папка `uploads/` добавлена в `.gitignore` - файлы не будут коммититься в Git.

## Миграция на объектное хранилище

В будущем можно легко мигрировать на:
- AWS S3
- Google Cloud Storage
- MinIO (для локальной разработки)
- Azure Blob Storage

При миграции нужно будет обновить только путь к файлам в БД.
